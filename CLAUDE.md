# dev-all-projects — 全プロジェクト共通ルール

> 各サブモジュールから `@../CLAUDE.md` で参照すること。

## トークン効率・コンテキスト管理

- **CLAUDE.md は 200行・25KB 以下に維持する**。溢れる情報は `docs/` に外部化し `@docs/xxx.md` で参照。
- セッション開始時は必ず `@docs/tasks.md` を確認し、実装済み・未実装を把握してから作業を開始すること。
- 実装が進んだら都度 `docs/tasks.md` に進捗を反映すること。次セッションで「`tasks.md` の未完了分から進めて」と伝えるだけで済むようになる。
- 指示を出す際は `@path/to/file` 形式でファイルを指定すること。プロジェクト全体スキャンによるトークン消費を防ぐ。

## 実装プロセス

- **3ファイル以上の変更・設計判断が必要な場合は `/plan` モードで計画を提示してから実装すること**。
- 大規模タスクは小さなステップに分割し、1ステップずつ承認を得て進めること。
- 曖昧な指示には、対象ファイル・関数名を特定するよう確認を求めること（試行錯誤によるトークン消費を防ぐ）。

## コマンド・ツール運用

- 会話が長くなり精度低下を感じたら `/compact` をユーザーに提案すること。
- 繰り返し発生する操作はスクリプト化（Shell/Python）を提案すること。
- モデルの使い分け：複雑な設計 → **Opus**、通常開発 → **Sonnet**、定型・反復作業 → **Haiku**。
- RTK（Rust Token Killer）がインストール済みの場合、冗長なCLI出力が予想されるコマンドは `rtk <cmd>` 経由で実行すること（例：`rtk git log`、`rtk npm test`）。
- `rtk init -g` によるグローバルフックが有効な場合、Bashツール呼び出しは自動的にRTK経由になる。明示的な `rtk` プレフィックスは不要。
- RTKはBashツール呼び出しのみに作用する。`Read`・`Grep`・`Glob` などのClaude Code組み込みツールはフックを経由しないため、シェルコマンド（`cat`/`grep`/`find`）または `rtk read`・`rtk grep`・`rtk find` を使うこと。

## Git・サブモジュール安全運用

- **`git push --force` および `--force-with-lease` は、全内容が他の回収可能な場所に存在することを明示的に確認するまで絶対に提案しないこと。**
- サブモジュールの変更は当該サブモジュール内でコミット後、親リポジトリでサブモジュール参照を更新すること。

## ユーザーへのフィードバック方針

- 誤字修正・軽微なドキュメント変更など、エディタで直接行った方が早いと判断される場合は、その旨をユーザーに伝えること。
- コマンド出力が大量になる場合はRTK経由での実行を優先すること。
- RTKがインストール済みであれば `rtk gain` で削減量を定期的に確認し、効果をユーザーに共有すること。

## セキュリティ

- APIキー・シークレットは絶対にチャットUIに貼り付けないこと。`~/.claude/settings.json` を直接編集すること。
- `.env` ファイルは `.gitignore` に含まれていることを常に確認すること。
